# JAMIESSHOESS Website

## How to run locally

**Easy way — double-click:**
```
START-SERVER.bat
```
Then open Chrome to: **http://localhost:8080**

**Manual way:**
```bash
npx serve . -p 8080
```

> ⚠️ Always use http://localhost:8080 — NOT file://
> Particle effects and some features only work on localhost or live domain.

## Clover Setup

### Step 1 — Get your tokens
1. Log in at [dashboard.clover.com](https://dashboard.clover.com)
2. Go to **Accounts & Setup → Ecommerce API Tokens**
3. Copy: Merchant ID, Private API Token, Ecommerce Secret Key, Ecommerce Public Key

### Step 2 — Configure
Edit `.env`:
```
CLOVER_MERCHANT_ID=A1B2C3D4E5
CLOVER_API_TOKEN=your_private_token
CLOVER_ECOM_SECRET_KEY=your_ecom_secret
CLOVER_ENV=production
PORT=3001
```

Edit `config.js`:
```js
backendUrl: 'https://your-app.up.railway.app',
cloverPublicKey: 'your_ecom_public_key',
```

### Step 3 — Run backend
```bash
npm install
node clover-sync.js
```

Backend starts on port 3001. Inventory syncs from Clover every 15 minutes.

## Deployment

### Backend — Railway (free)
1. [railway.app](https://railway.app) → New Project → Deploy from GitHub
2. Set environment variables (same as `.env`)
3. Copy the generated URL → paste into `config.js` as `backendUrl`

### Frontend — GitHub Pages
1. Push to GitHub
2. Settings → Pages → Source: master branch
3. Live at `https://kandonmcguirk72.github.io/jamiesshoess-site`

## How it works

| What | Where |
|------|-------|
| Inventory sync | `clover-sync.js` reads Clover API → writes `data/inventory.json` every 15 min |
| Product display | `app.js` fetches `data/inventory.json` on page load |
| Checkout | Cart → Clover card iframe → POST `/checkout` → charge |
| Stock update | Successful sale decrements stock in `inventory.json` AND in Clover POS |

If Clover is not configured (`config.js` left blank), falls back to built-in catalog + Instagram DM checkout.
